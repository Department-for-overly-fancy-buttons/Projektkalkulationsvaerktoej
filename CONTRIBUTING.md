## Asking Questions
Asking questions should be done through filing bugs or featuring requests. If it's urgent, feel free to write a mail to DepartmentOfOverlyFancyButtons@DiffenatelyNotAfakeMail.com, which we can guarantee with a 20% certainty is not a fake mail. 

## Opening an Issue
Make sure you have the latest version of the project before [creating an issue](https://help.github.com/en/github/managing-your-work-on-github/creating-an-issue). In many cases, this will fix the issue. If not, follow Githubs guide linked here to open the issue.

## Bug Reports and Other Issues
To contribute the best way possible, please follow these steps to make the best possible bug-report! We truely appriciate the bug-report being written as clearly as possible, as it will help us fix the issue faster.
#### Steps:
- **Make sure you do not open a duplicate issue**. Check if the issue already exists
- **Do not comment on existing issues**, if you just want to +1 it. **Instead, do [reactions](https://github.blog/2016-03-10-add-reactions-to-pull-requests-issues-and-comments/)**
- **Be clear with the bug.** Be concise and descriptive. Provide all the information you possibly can, and if possible, include the steps to reproduce the bug. It will help a lot to fix the issue.
